# Application Design — trigger × キャラクター × トーン 組み合わせ定義

## アプリの特性を踏まえた設計方針

DAMESIは「人をダメにする」アプリであり、ユーザーの思考停止を心地よく肯定する体験を提供する。
キャラクターの一言は以下の目的を持つ：
- **毎回違うキャラクターが出るランダム性**で飽きない体験
- **トーンの使い分け**でユーザーの行動を肯定・応援・笑わせる
- **堕落ルートのみ固定キャラ**（メシストフェレス）で演出のメリハリ

## 提案: `meal_suggested` の発火タイミングと表示方式

**発火タイミング**: フィルタリング画面が表示されたとき

**表示方式**: ボトムシートではなく、**フィルタリング画面内にインライン表示**
- フィルタリング画面の上部にキャラクター画像 + キャラクター固有の質問文言を表示
- stories.md US-05 で既に定義済みの「キャラクター別の質問文言」がこれに該当
- CharacterBottomSheet は使用しない（画面遷移を伴わないため）

**`gacha_decided` の表示方式**: `meal_decided` と同じ **ボトムシート表示**
- ガチャ確定時もボトムシートでキャラクターの一言を表示
