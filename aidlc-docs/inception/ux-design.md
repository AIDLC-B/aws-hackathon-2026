# UX設計ドキュメント

## 基本方針

- **最頻利用機能**: 献立提案（US-05）を中心に設計
- **トップ画面設計思想**: 状況適応型（献立の確定状態によって表示を切り替える）
- **ナビゲーション**: ボトムナビゲーション（タブバー）

---

## ナビゲーション構成

```
+---------------------------+
|                           |
|   （各画面のコンテンツ）   |
|                           |
+---------------------------+
| 🏠 ホーム | 🍳 レシピ | ⚙ 設定 |
+---------------------------+
```

| タブ | 内容 |
|---|---|
| 🏠 ホーム | 献立フロー全体（選択→提案→ガチャ→確定） |
| 🍳 レシピ | 料理登録・レパートリー一覧・編集（US-01, US-04） |
| ⚙ 設定 | ニックネーム表示・推しキャラ設定（プレミアム）・ログアウト |

---

## トップ画面の分岐ロジック

献立の確定状態によって表示を切り替える（セッション単位ではなく献立確定状態で管理）。

```
アプリを起動
    |
    +-- 未ログイン ──────────→ 【ログイン画面】
    |                               ↓
    |                         ログイン成功
    |                               ↓
    +-- 初回ログイン ─────────→ 【オンボーディング画面】
    |                               ↓
    |                         料理を選択 or スキップ
    |                         （バナナは強制追加）
    |                               ↓
    |                         【選択画面】へ
    |
    +-- 2回目以降（ログイン済み）
            |
            +-- 献立が0件 ──────────→ 【選択画面】
            |
            +-- 献立が1件確定済み ───→ 【レシピ詳細画面】（直接表示）
            |
            +-- 献立が2件以上確定済み→ 【献立リスト画面】
```

## 画面設計

### 【オンボーディング画面】（初回起動時のみ）

```
+---------------------------+
|  [サボ母ちゃんのアイコン]  |
|  「あんた、はじめてやね！  |
|   作れる料理、選んでみ？   |
|   バナナは絶対入れといたで」|
|                           |
|  ☑ 🍌 バナナ              |
|     よく作る｜かんたん｜1分 |  ← 選択済み・変更不可
|  □ 🥚 卵かけご飯（TKG）   |
|     よく作る｜かんたん｜2分 |
|  □ 🍛 カレー（レトルト）  |
|     よく作る｜かんたん｜10分|
|  ...（20件）              |
|                           |
|  [はじめる！（1件選択中）] |
+---------------------------+
```

- サボ母ちゃんが出迎えてキャラクターを最初に印象づける
- バナナは最初から選択済み・チェックを外せない
- 各料理に絵文字・料理名・作る頻度・難易度・所要時間を表示
- 選択数は0件でも「はじめる！」を押せる
- 2回目以降は表示されない（初回フラグで管理）

---

- 献立が未確定のときに表示
- 2択のみ（料理登録は含めない）
- コピーは日付を意識しない言葉を使用

```
+---------------------------+
|  DAMESI                   |
|                           |
|   さて、どうする？         |
|  （※コピーは要検討）       |
|                           |
|  +---------------------+  |
|  |  😤 考えたくない     |  |
|  +---------------------+  |
|                           |
|  +---------------------+  |
|  |  🎰 運に任せる       |  |
|  +---------------------+  |
|                           |
+---------------------------+
```

**コピー候補**:
- 「さて、どうする？」
- 「献立、任せる？」
- 「考える？それとも任せる？」

### 【レシピ詳細画面】（献立1件確定済み）

```
+---------------------------+
|  今の献立                  |
|                           |
|  [料理写真]                |
|                           |
|  🍛 カレーライス  SR       |
|  難易度: ふつう            |
|  所要時間: 30分            |
|                           |
|  材料                     |
|  （未登録の場合は「未設定」）|
|                           |
|  レシピ                   |
|  （未登録の場合は「未設定」）|
|                           |
|  メモ                     |
|  （未登録の場合は「未設定」）|
|                           |
|  [✏️ 編集]  [✅ つくったよ！]|
+---------------------------+
```

- 写真・料理名・所要時間・材料・レシピ・メモを表示
- 材料・レシピ・メモが未登録の場合は「未設定」と表示
- 「編集」ボタンからレシピ編集画面へ遷移
- 「つくったよ！」ボタンでUS-16フローへ

### 【レシピ編集画面】

```
+---------------------------+
|  レシピを編集する          |
|                           |
|  [写真エリア]              |
|  （タップで変更）          |
|                           |
|  料理名 *                 |
|  [________________]       |
|                           |
|  頻度 *                   |
|  [よく作る ▼]             |
|                           |
|  難易度 *                 |
|  [かんたん ▼]             |
|                           |
|  所要時間 *               |
|  [__ 分]                  |
|                           |
|  材料                     |
|  [________________]       |
|                           |
|  レシピ                   |
|  [________________]       |
|  （複数行テキスト）        |
|                           |
|  メモ                     |
|  [________________]       |
|                           |
|  [保存する]               |
|  [🗑 この料理を削除する]   |
+---------------------------+
```

- 料理名・頻度・難易度・所要時間は必須（バリデーションあり）
- 材料・レシピ・メモは任意入力
- US-01の登録画面と同じバリデーションルールを適用
- 「保存する」後はレシピ詳細画面へ戻る
- 「この料理を削除する」押下 → 確認ダイアログ（「削除すると元に戻せません。削除しますか？」）→ 確認後レパートリーから削除 → レパートリー一覧へ遷移
- レシピ登録画面（US-01）とレシピ編集画面は同じ入力項目を持つ（材料・レシピ・メモの有無のみ異なる）

### 【献立リスト画面】（献立2件以上・主に10連ガチャ後）

```
+---------------------------+
|  DAMESI                   |
|                           |
|  今の献立  (10件)          |
|  ┌────────────────────────┐|
|  │ 🍛 カレーライス  SR    │|
|  │              [✅ つくった]│|
|  └────────────────────────┘|
|  ┌────────────────────────┐|
|  │ 🥚 卵かけご飯     N    │|
|  │              [✅ つくった]│|
|  └────────────────────────┘|
|  ┌────────────────────────┐|
|  │ 🍖 唐揚げ        SR    │|
|  │              [✅ つくった]│|
|  └────────────────────────┘|
|        ...                |
+---------------------------+
```

- 各行タップ → レシピ詳細画面へ
- 「つくった」ボタン押下 → キャラクターの一言（褒めトーン）→ その1件をリストから削除
- 削除後に件数が1件になれば詳細画面へ、0件になれば選択画面へ自動遷移
- 直接削除ボタン（🗑）は不要

### 全体フロー

```
【アプリ起動時】
    |
    +-- 未ログイン ──→ 【ログイン画面】
    |                   ├─ ログイン成功（初回）→ オンボーディング → 選択画面
    |                   ├─ ログイン成功（2回目以降）→ トップ画面
    |                   └─ 「新規登録」→ 【サインアップ画面】
    |                                    → 確認コード入力 → オンボーディング
    |
    +-- ログイン済み → トップ画面（献立状態で分岐）

【🏠 ホームタブ内のフロー】

トップ画面
    ├─ 献立0件   → 選択画面
    │               ├─ 「考えたくない」（10件以上時）→ フィルタリング画面
    │               │       └→ 3品提案画面 → 確定 → ボトムシート → レシピ詳細画面
    │               └─ 「運に任せる」 → ガチャ画面
    │                       ├─ シングルガチャ → 確定 → ボトムシート → レシピ詳細画面
    │                       └─ 10連ガチャ   → 確定 → ボトムシート → 献立リスト画面
    │
    ├─ 献立1件   → レシピ詳細画面
    │               ├─ 「編集」        → レシピ編集画面
    │               │                   ├─ 「保存する」         → レシピ詳細画面
    │               │                   └─ 「この料理を削除する」→ 確認ダイアログ
    │               │                                              → レパートリー一覧
    │               └─ 「つくったよ！」→ ボトムシート → 削除 → 選択画面（0件）
    │
    └─ 献立2件以上 → 献立リスト画面
                    ├─ 行タップ        → レシピ詳細画面（戻る→献立リスト画面）
                    │                   ├─ 「編集」        → レシピ編集画面
                    │                   └─ 「つくったよ！」→ ボトムシート → 削除 → 件数で分岐
                    └─ 「つくった」    → ボトムシート → 削除 → 件数で分岐
                                          ├─ 0件 → 選択画面
                                          ├─ 1件 → レシピ詳細画面
                                          └─ 2件以上 → 献立リスト画面（更新）

【🍳 レシピタブ内のフロー】

レパートリー一覧
    ├─ 「+ 登録」  → 料理登録画面 → 登録完了 → ボトムシート → 料理登録画面
    └─ 行タップ   → レシピ詳細画面（戻る→レパートリー一覧）
                    └─ 「編集」   → レシピ編集画面
                                    ├─ 「保存する」         → レシピ詳細画面
                                    └─ 「この料理を削除する」→ 確認ダイアログ
                                                               → レパートリー一覧
```

## 献立の状態管理ルール

| 操作 | 登録件数の変化 | キャラクター一言 |
|---|---|---|
| ワンボタン提案で確定 | +1件 | あり（確定時） |
| シングルガチャで確定 | +1件 | あり（確定時） |
| 10連ガチャで確定（献立0件時） | +10件 | あり（確定時） |
| 10連ガチャで確定（追加選択時） | +10件（既存に追加） | あり（確定時） |
| 10連ガチャで確定（入れ替え選択時） | 既存削除 → 10件 | あり（確定時） |
| 「つくったよ！」ボタン押下 | -1件 | あり（褒めトーン） |
| 削除後0件になった場合 | → 選択画面へ遷移 | なし |

## 未検討事項（次回議論候補）

- 【フィルタリング画面】の質問数・形式（「考えたくない」選択後）→ US-05で確定済み
- 【確定画面】のキャラクター一言の見せ方 → US-13で確定済み
- 選択画面のコピー確定

## キャラクター一言ボトムシート仕様（確定）

- **表示形式**: 画面下部からスライドアップ（ボトムシート）
- **内容**: キャラクター画像 + キャラクター名 + 一言セリフ
- **閉じ方**: 「閉じる」ボタン または 5秒後に自動クローズ
- **発火タイミング**:
  - 献立確定時（提案・ガチャ問わず）: trigger=meal_decided
  - 「つくったよ！」押下時: trigger=meal_completed
  - 料理登録完了時: trigger=recipe_registered
