# ユーザーストーリー

**ペルソナ**: 田中 さおり（主婦・主夫ユーザー）  
**優先度**: MoSCoW法（Must / Should / Could / Won't）  
**Acceptance Criteria形式**: Given / When / Then（BDD形式）

---

## ドメイン 0: 認証（Cognito）

### US-17: サインアップ・ログインする
**As a** 新規・既存ユーザーとして  
**I want to** メールアドレスとパスワードでアカウントを作成・ログインしたい  
**So that** 自分のレパートリーと献立データを安全に管理できる  

**優先度**: Must

**画面レイアウト（ログイン画面）**:
```
+---------------------------+
|  DAMESI                   |
|                           |
|  メールアドレス            |
|  [________________]       |
|                           |
|  パスワード               |
|  [________________]       |
|                           |
|  [ログイン]               |
|                           |
|  アカウントをお持ちでない方 |
|  [新規登録はこちら]        |
+---------------------------+
```

**画面レイアウト（サインアップ画面）**:
```
+---------------------------+
|  アカウント作成            |
|                           |
|  ニックネーム *            |
|  [________________]       |
|                           |
|  メールアドレス *          |
|  [________________]       |
|                           |
|  パスワード *              |
|  [________________]       |
|                           |
|  [登録する]               |
|                           |
|  すでにアカウントをお持ちの方|
|  [ログインはこちら]        |
+---------------------------+
```

**Acceptance Criteria**:

```
【アプリ起動時の認証チェック】

Given ユーザーがアプリを起動した
When 未ログイン状態の場合
Then ログイン画面が表示される（オンボーディング・ホーム画面は表示されない）

Given ユーザーがアプリを起動した
When ログイン済み（セッション有効）の場合
Then ログイン画面をスキップしてトップ画面へ遷移する（自動ログイン維持）

【ログインフロー】

Given ログイン画面が表示されている
When メールアドレスとパスワードを入力して「ログイン」を押す
Then Cognitoで認証が行われる

Given 認証が成功した
When ログイン処理が完了する
Then 初回ログインの場合はオンボーディング画面へ遷移する
Then 2回目以降はトップ画面へ遷移する

Given 認証が失敗した
When メールアドレスまたはパスワードが誤っている
Then 「メールアドレスまたはパスワードが正しくありません」と表示される
Then パスワードフィールドがクリアされる

【サインアップフロー】

Given ログイン画面で「新規登録はこちら」を押した
When サインアップ画面が表示される
Then ニックネーム・メールアドレス・パスワードの入力フォームが表示される

Given サインアップ画面で全項目を入力して「登録する」を押した
When Cognitoにアカウントが作成される
Then 入力したメールアドレスに確認コードが送信される
Then 確認コード入力画面へ遷移する

Given 確認コード入力画面が表示されている
When 正しい確認コードを入力して「確認」を押す
Then アカウントが有効化される
Then 自動的にログインされてオンボーディング画面へ遷移する

Given 確認コード入力画面が表示されている
When 誤った確認コードを入力して「確認」を押す
Then 「確認コードが正しくありません」と表示される

Given 確認コードが届かなかった
When 「コードを再送する」を押す
Then 新しい確認コードがメールアドレスに送信される

【バリデーション（サインアップ）】

Given サインアップ画面で「登録する」を押した
When ニックネームが未入力の場合
Then 「ニックネームは必須です」と表示され登録がブロックされる

Given サインアップ画面で「登録する」を押した
When メールアドレスの形式が不正な場合
Then 「正しいメールアドレスを入力してください」と表示される

Given サインアップ画面で「登録する」を押した
When パスワードがCognitoのポリシーを満たさない場合
Then 「パスワードは8文字以上で入力してください」と表示される
```

### US-18: ログアウトする
**As a** ログイン済みユーザーとして  
**I want to** 設定画面からログアウトしたい  
**So that** 別のアカウントに切り替えたり、アカウントを保護できる  

**優先度**: Should

**Acceptance Criteria**:

```
Given ユーザーが設定タブを開いた
When 設定画面が表示される
Then 「ログアウト」ボタンが表示される

Given 設定画面で「ログアウト」を押した
When 確認ダイアログが表示される
Then 「ログアウトしますか？」というメッセージと
     「ログアウト」「キャンセル」ボタンが表示される

Given 確認ダイアログで「ログアウト」を押した
When ログアウト処理が完了する
Then Cognitoのセッションが破棄される
Then ログイン画面へ遷移する
Then ローカルのキャッシュデータはクリアされる
```

## ドメイン 1: 料理管理（FR-01）

### US-00: 初回オンボーディングでレパートリーを作る
**As a** 初めてDAMESIを使う主婦ユーザーとして  
**I want to** アプリ初回起動時に簡単な料理リストから作れるものを選びたい  
**So that** 最初からレパートリーが揃った状態でDAMESIを使い始められる  

**優先度**: Must

**Acceptance Criteria**:

```
Given ユーザーがアプリを初めて起動した
When オンボーディング画面が表示される
Then サボ母ちゃんのアイコンと歓迎セリフが表示される
Then 20件の料理リストが表示される（絵文字・料理名・作る頻度・難易度・所要時間）
Then バナナは選択済み・チェックを外せない状態で表示される
Then ボタンに「はじめる！（1件選択中）」と現在の選択件数が表示される

Given オンボーディング画面でユーザーが料理を選択・解除した
When 選択状態が変わる
Then ボタンの選択件数表示がリアルタイムで更新される

Given オンボーディング画面が表示されている
When ユーザーが「はじめる！」を押す
Then 選択した料理（バナナ含む）がデフォルト値でレパートリーに追加される
Then キャラクターの一言（褒めトーン・trigger=recipe_registered, from=onboarding）がボトムシートで表示される
Then ボトムシートを閉じると選択画面へ遷移する

Given ユーザーが2回目以降にアプリを起動した
When アプリが起動する
Then オンボーディング画面は表示されない
```

### US-01: 料理を登録する
**As a** 主婦ユーザーとして  
**I want to** 料理の写真を撮るか手動入力するだけで、料理をレパートリーに登録したい  
**So that** 手間なく自分が作れる料理を蓄積できる  

**優先度**: Must

**画面レイアウト（確認・編集画面）**:
```
+---------------------------+
|  料理を登録する            |
|                           |
|  [サボ母ちゃん]            |
|  「料理名と頻度だけ入れて  |
|   くれたらええで！材料や   |
|   レシピはあとでゆっくり   |
|   入れたらええねん」       |
|                           |
|  [写真エリア]              |
|  （タップで撮影/選択）     |
|                           |
|  料理名 *                 |
|  [________________]       |
|                           |
|  頻度 *                   |
|  [よく作る ▼]             |
|                           |
|  難易度 *                 |
|  [かんたん ▼]             |
|                           |
|  所要時間 *               |
|  [__ 分]                  |
|                           |
|  [登録する]               |
+---------------------------+
```

**Acceptance Criteria**:

```
【写真ありフロー】

Given ユーザーがレシピタブから料理登録画面を開いた
When 画面が表示される
Then 写真エリア・料理名・頻度・難易度・所要時間の入力フォームが同一画面に表示される
Then サボ母ちゃんのアイコンと「材料やレシピはあとから入力できる」旨の説明セリフが表示される
Then 全項目に「*（必須）」の表示がある

Given ユーザーが写真エリアをタップした
When カメラ/ライブラリ選択ダイアログが表示される
Then 「撮影する」「ライブラリから選ぶ」の2択が表示される

Given ユーザーが写真を選択・撮影した
When AIが写真を解析する
Then 料理名・難易度・所要時間・頻度が自動入力される
Then ユーザーは各項目を確認・修正できる

Given AIが写真から料理を認識できなかった
When 解析が完了する
Then 「料理を認識できませんでした。手動で入力してください」というメッセージが表示される
Then 全項目が空欄のまま手動入力できる状態になる

【手動入力フロー】

Given ユーザーが写真を追加せずに各項目を入力した
When 料理名・頻度・難易度・所要時間をすべて入力して「登録する」を押す
Then 料理がレパートリーに追加される

【バリデーション】

Given 「登録する」ボタンを押した
When 料理名が未入力の場合
Then 料理名フィールドの下に「料理名は必須です」と表示され登録がブロックされる

Given 「登録する」ボタンを押した
When 頻度が未選択の場合
Then 頻度フィールドの下に「頻度は必須です」と表示され登録がブロックされる

Given 「登録する」ボタンを押した
When 難易度が未選択の場合
Then 難易度フィールドの下に「難易度は必須です」と表示され登録がブロックされる

Given 「登録する」ボタンを押した
When 所要時間が未入力の場合
Then 所要時間フィールドの下に「所要時間は必須です」と表示され登録がブロックされる

【登録完了後】

Given すべての必須項目が入力されて「登録する」を押した
When 登録が完了する
Then キャラクターの一言（褒めトーン・US-14）が表示される
Then 一言を閉じると料理登録画面に戻る（続けて登録できる）
```

**レアリティの表示ルール**:
- **登録画面・レパートリー一覧**: 「よく作る / しばしば作る / たまに作る / まれに作る」と表示
- **ガチャ演出・ガチャ結果画面**: 「N / R / SR / SSR」と表示（ゲーム的な演出として）

**スコープ外（あとから設定）**:
- 材料リスト
- レシピ・調理手順
- メモ・コメント

### US-04: 登録済み料理を確認・編集する
**As a** 主婦ユーザーとして  
**I want to** 登録済みのレパートリーを一覧で確認・編集したい  
**So that** 古い情報を更新したり、材料・レシピ・メモを追加できる  

**優先度**: Should

**Acceptance Criteria**:

```
Given ユーザーがレシピタブを開いた
When 画面が表示される
Then 登録済みの料理が一覧表示される（料理名・頻度・難易度・所要時間）

Given レパートリー一覧で料理をタップした
When レシピ詳細画面が表示される
Then 写真・料理名・所要時間・材料・レシピ・メモが表示される
Then 材料・レシピ・メモが未登録の場合は「未設定」と表示される

Given レシピ詳細画面で「編集」を押した
When レシピ編集画面が表示される
Then 料理名・頻度・難易度・所要時間・材料・レシピ・メモが編集できる
Then 料理名・頻度・難易度・所要時間は必須（未入力時は各フィールド下にエラー表示）
Then 材料・レシピ・メモは任意入力

Given 編集画面で内容を変更して「保存する」を押した
When 保存が完了する
Then レシピ詳細画面へ戻り、更新された内容が表示される

Given レシピ編集画面で「この料理を削除する」を押した
When 確認ダイアログが表示される
Then 「削除すると元に戻せません。削除しますか？」というメッセージと
     「削除する」「キャンセル」ボタンが表示される

Given 確認ダイアログで「削除する」を押した
When 削除が完了する
Then レパートリーから料理が削除されレパートリー一覧へ遷移する
```

## ドメイン 2: 献立提案（FR-02, FR-04）

### US-05: 「考えたくない」を選んで献立を提案してもらう
**As a** 主婦ユーザーとして  
**I want to** 選択画面で「考えたくない」を選ぶだけで献立候補を3品提案してもらいたい  
**So that** 何を作るか考える時間をゼロにできる  

**優先度**: Must

**フィルタリング画面レイアウト**:
```
+---------------------------+
|  [ランダムキャラクター]    |
|  （キャラ別の質問文言）    |
|                           |
|  気分は？                 |
|  [😤 なんでもいい]         |
|  [🌟 ちょっとがんばる]     |
|  [😴 楽したい ✓]          |  ← デフォルト選択済み
|                           |
|  所要時間は？             |
|  [15分以内 ✓][30分以内]   |  ← デフォルト選択済み
|  [気にしない]              |
|                           |
|  難易度は？               |
|  [かんたん ✓][ふつう]     |  ← デフォルト選択済み
|  [がんばる][なんでもいい]  |
|                           |
|  [この条件で探す！]        |
+---------------------------+
```

**キャラクター別の質問文言例**:

| キャラクター | 質問文言 |
|---|---|
| サボエル | 「かつて私も献立に悩みました…条件を教えてください」 |
| サボ母ちゃん | 「ほな、今日はどんな感じにする？正直に言いや！」 |
| ニャマケ | 「…えっと…何が食べたい…？…zzz」 |
| シェフレイ | 「条件を…述べよ…私が…選んでやる…」 |
| メシストフェレス | 「ふふ、どんな条件でも私が最高の選択をしてあげるよ」 |
| サボット | 「入力してください。最適解を算出します。QED。」 |
| サボわらし | 「えへへ、どんなごはんがいい〜？」 |

**Acceptance Criteria**:

```
【フィルタリング画面表示】

Given ユーザーが選択画面で「😤 考えたくない」を押した
When フィルタリング画面が表示される
Then 7キャラクターからランダムで1体が選ばれ、そのキャラクターの画像と
     キャラクター固有の質問文言が表示される
Then 気分・所要時間・難易度の3つの質問が1画面に並んで表示される
Then 各質問の選択肢ボタンが表示される
     気分: 「😤 なんでもいい」「🌟 ちょっとがんばる」「😴 楽したい」
     所要時間: 「15分以内」「30分以内」「気にしない」
     難易度: 「かんたん」「ふつう」「がんばる」「なんでもいい」
Then デフォルトで以下が選択済み状態になっている
     気分: 「😴 楽したい」
     所要時間: 「15分以内」
     難易度: 「かんたん」

【条件選択】

Given フィルタリング画面が表示されている
When ユーザーが各質問の選択肢をタップする
Then タップした選択肢がハイライト（選択済み）状態になる
Then 各質問につき1つのみ選択できる

Given フィルタリング画面が表示されている
When 「この条件で探す！」を押す
Then 選択した条件でレパートリーを絞り込む
     （気分=楽したい → 難易度:かんたんを優先、所要時間=15分以内 → 15分以下のみ）

【3品以上ある場合】

Given 絞り込み結果が3品以上ある
When フィルタリングが完了する
Then 絞り込み結果からランダムで3品が選ばれて提案画面に表示される
Then 各提案カードに料理名・頻度・難易度・所要時間が表示される

Given 3品の提案が表示された
When ユーザーが1品をタップして「これにする！」を押す
Then 献立が確定し、確定画面（キャラクターの一言・US-13）に遷移する

Given 確定画面を閉じた
When トップ画面に戻る
Then 献立が1件登録された状態でレシピ詳細画面が表示される

【3品未満の場合】

Given 絞り込み結果が1〜2品しかない
When フィルタリングが完了する
Then 残った1〜2品が提案画面に表示される
Then 7キャラクターからランダムで1体が選ばれ、キャラクター固有の
     ガチャ誘導セリフとともに「ガチャで決める？」ボタンが表示される
     （例）サボ母ちゃん:「料理が少ないなあ…ガチャに任せてみいや！」
     （例）メシストフェレス:「ふふ、こっちにおいでよ…ガチャの方が楽しいよ？」
     （例）サボット:「候補数が不足しています。ガチャ移行を推奨します。QED。」

Given 3品未満の提案画面で「ガチャで決める？」を押した
When ガチャ画面へ遷移する
Then シングルガチャ画面が表示される（US-10へ合流）

Given 3品未満の提案画面で残った料理から「これにする！」を押した
When 献立が確定する
Then 確定画面（キャラクターの一言・US-13）に遷移する

【絞り込み結果が0件の場合】

Given 絞り込み結果が0件
When フィルタリングが完了する
Then 「条件に合う料理が見つかりませんでした」と表示される
Then キャラクターが「ガチャで決める？」ボタンとともにガチャ誘導セリフを表示する
```

### US-06: レパートリーが少ないときガチャへ誘導される
**As a** 主婦ユーザーとして  
**I want to** レパートリーが少ないときに適切な案内をしてほしい  
**So that** 迷わず次のアクションに進める  

**優先度**: Must

**Acceptance Criteria**:

```
【選択画面でのガチャ誘導】

Given ユーザーのレパートリーが10件未満の状態で
When 選択画面が表示される
Then 「😤 考えたくない」ボタンはグレーアウトして押せない状態になる
Then 7キャラクターからランダムで1体が選ばれ、キャラクター固有の
     レパートリー不足セリフとともに「🎰 運に任せる」ボタンのみが有効で表示される
     （例）サボ母ちゃん:「まだ料理が少ないなあ。ガチャで遊んでみいや！」
     （例）サボット:「レパートリー数が閾値未満です。ガチャを推奨します。QED。」
     （例）ニャマケ:「…料理少ない…ガチャ…しよ…zzz」

Given 選択画面でレパートリーが10件未満の状態で
When ユーザーが「🎰 運に任せる」を押す
Then ガチャ画面へ遷移する（US-10へ合流）

【レパートリーが10件以上になった場合】

Given ユーザーがレパートリーを登録して10件以上になった
When 選択画面が表示される
Then 「😤 考えたくない」ボタンが有効になる
Then キャラクターのガチャ誘導セリフは表示されない
```

## ドメイン 3: 献立ガチャ（FR-05）

### US-10: シングルガチャで献立を一発決定する
**As a** 主婦ユーザーとして  
**I want to** ガチャ演出で今日の献立を1品決めてもらいたい  
**So that** 意思決定を完全に放棄して楽しめる  

**優先度**: Must

**画面レイアウト（ガチャ画面）**:
```
+---------------------------+
|  献立ガチャ               |
|                           |
|  +---------------------+  |
|  |  [カプセルトイ画像]  |  |
|  |  （回転アニメーション）|  |
|  +---------------------+  |
|                           |
|  [🎰 ガチャを回す！]       |
|  [10連ガチャ]             |
|                           |
+---------------------------+
```

**Acceptance Criteria**:

```
【ガチャ画面表示】

Given ユーザーが選択画面で「🎰 運に任せる」を押した
When ガチャ画面が表示される
Then カプセルトイのイラスト画像が表示される
```

### US-13: 献立確定時にキャラクターの一言を受け取る
**As a** 主婦ユーザーとして

**So that** 毎回の操作が楽しくなり、アプリを使い続けたくなる  

**優先度**: Must

**表示形式（ボトムシート）**:
```
+---------------------------+
|  （背景：確定した献立画面）|
|                           |
|                           |
+---------------------------+
|  [キャラクター画像]        |  ← 画面下部からスライドアップ
|  「キャラクター固有セリフ」 |
|                           |
|  [閉じる]  （5秒で自動）   |
+---------------------------+
```

**Acceptance Criteria**:

```
【ボトムシート表示】

Given ユーザーが献立を確定した
When 確定処理が完了する
Then 画面下部からボトムシートがスライドアップして表示される
Then ボトムシートにキャラクター画像・キャラクター名・一言セリフが表示される
Then 「閉じる」ボタンが表示される
Then 5秒後に自動でボトムシートが閉じる

Given ボトムシートが表示されている
When ユーザーが「閉じる」ボタンを押す
Then ボトムシートが閉じる（自動クローズのタイマーはキャンセルされる）

【キャラクター・トーンの決定】

When キャラクタードメインにtrigger・fromを渡す
Then キャラクタードメイン内部でキャラクター・トーンを決定する
     （呼び出し元はキャラクター・トーンを指定しない）
Then 同じ状況で複数回確定しても毎回同じキャラクターが出るとは限らない

【trigger・fromの値】

| 状況 | trigger | from |
|---|---|---|
| 提案から確定 | meal_decided | suggestion |
| シングルガチャから確定 | gacha_decided | gacha |
| 10連ガチャから確定 | gacha_decided | gacha_10 |

【ボトムシート閉じた後の遷移】

Given ボトムシートが閉じた（自動・手動問わず）
When 提案フロー（from: suggestion）から確定した場合
Then レシピ詳細画面が表示される（献立1件）

Given ボトムシートが閉じた
When シングルガチャ（from: gacha）から確定した場合
Then レシピ詳細画面が表示される（献立1件）

Given ボトムシートが閉じた
When 10連ガチャ（from: gacha_10）から確定した場合
Then 献立リスト画面が表示される（献立10件）
```
