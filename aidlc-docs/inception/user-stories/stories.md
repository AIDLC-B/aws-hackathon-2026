# ユーザーストーリー

**ペルソナ**: 田中 さおり（主婦・主夫ユーザー）  
**優先度**: MoSCoW法（Must / Should / Could / Won't）  
**Acceptance Criteria形式**: Given / When / Then（BDD形式）

---

## ドメイン 1: 料理管理（FR-01）

### US-01: 写真で料理を登録する
**As a** 主婦ユーザーとして  
**I want to** 作った料理の写真をアップロードするだけで料理を登録したい  
**So that** 手入力の手間なく自分のレパートリーを増やせる  

**優先度**: Must

**Acceptance Criteria**:

```
Given ユーザーが料理写真をアップロードした
When AIが写真を解析する
Then 料理名・難易度・所要時間が自動入力された確認画面が表示される

Given AI認識結果の確認画面が表示されている
When ユーザーが内容を確認して「登録する」を押す
Then 料理がレパートリーに追加される

Given AI認識結果の確認画面が表示されている
When ユーザーが料理名・難易度・所要時間を修正して「登録する」を押す
Then 修正後の内容でレパートリーに追加される
```

---

### US-02: 手動で料理を登録する
**As a** 主婦ユーザーとして  
**I want to** 写真なしで料理名・難易度・所要時間を手動入力して登録したい  
**So that** 写真がない料理もレパートリーに追加できる  

**優先度**: Must

**Acceptance Criteria**:

```
Given ユーザーが手動入力フォームを開いた
When 料理名・難易度・所要時間・レアリティを入力して「登録する」を押す
Then 料理がレパートリーに追加される

Given ユーザーが手動入力フォームを開いた
When 料理名を入力せずに「登録する」を押す
Then 「料理名は必須です」というエラーが表示され登録されない
```

---

### US-03: 料理登録時にレアリティを設定する
**As a** 主婦ユーザーとして  
**I want to** 料理を登録するときにN/R/SR/SSRのレアリティを自分で選びたい  
**So that** 「よく作る料理」と「たまにしか作らない料理」を自分の感覚で分類できる  

**優先度**: Must

**Acceptance Criteria**:

```
Given ユーザーが料理登録画面を開いた（写真あり・なし問わず）
When レアリティ選択欄が表示される
Then N（よく作る）/ R（しばしば作る）/ SR（たまに作る）/ SSR（まれに作る）の4択が表示される

Given レアリティ選択欄が表示されている
When ユーザーがレアリティを選択せずに「登録する」を押す
Then 「レアリティを選択してください」というエラーが表示される

Given ユーザーがレアリティを選択した
When 「登録する」を押す
Then 選択したレアリティで料理が登録される
```

---

### US-04: 登録済み料理を確認・編集する
**As a** 主婦ユーザーとして  
**I want to** 登録済みのレパートリーを一覧で確認・編集したい  
**So that** 古い情報を更新したり、不要な料理を削除できる  

**優先度**: Should

**Acceptance Criteria**:

```
Given ユーザーがレパートリー一覧画面を開いた
When 画面が表示される
Then 登録済みの料理が一覧表示される（料理名・レアリティ・難易度・所要時間）

Given レパートリー一覧画面で料理を選択した
When 「編集」を押す
Then 料理名・難易度・所要時間・レアリティを編集できる画面が表示される

Given 編集画面で内容を変更した
When 「保存」を押す
Then 変更内容が反映される
```

## ドメイン 2: 献立提案（FR-02, FR-04）

### US-05: ワンボタンで今日の献立を提案してもらう
**As a** 主婦ユーザーとして  
**I want to** ボタン1つで今日の献立候補を3品提案してもらいたい  
**So that** 何を作るか考える時間をゼロにできる  

**優先度**: Must

**Acceptance Criteria**:

```
Given ユーザーがトップ画面を開いた
When 「今日の献立を決めて！」ボタンを押す
Then 質問形式のフィルタリング画面が表示される（「今日の気分は？」「時間はある？」など）

Given フィルタリング質問に回答した
When 絞り込み結果が10件以内になった
Then レパートリーからランダムで3品が提案される

Given 3品の提案が表示された
When ユーザーが1品を選んで「これにする！」を押す
Then 献立が確定し、確定画面に遷移する
```

---

### US-06: レパートリーが少ないときガチャへ誘導される
**As a** 主婦ユーザーとして  
**I want to** レパートリーが少ないときに適切な案内をしてほしい  
**So that** 迷わず次のアクションに進める  

**優先度**: Must

**Acceptance Criteria**:

```
Given ユーザーのレパートリーが3品未満の状態で
When 「今日の献立を決めて！」ボタンを押す
Then 「まずはガチャを回してみよう！」というメッセージとガチャ画面へのボタンが表示される
```

---

### US-07: 「楽したい」モードで難易度を絞り込む
**As a** 主婦ユーザーとして  
**I want to** 「今日は疲れた」ときに簡単な料理だけを提案してもらいたい  
**So that** 無理せず作れる献立を選べる  

**優先度**: Should

**Acceptance Criteria**:

```
Given フィルタリング質問画面が表示されている
When ユーザーが「今日は楽したい」を選択する
Then 難易度「かんたん」の料理のみを対象にフィルタリングされる

Given 「かんたん」の料理が3品未満の場合
When フィルタリングを実行する
Then 「かんたんな料理が少ないです。ガチャで運試しをしますか？」と表示される
```

---

### US-08: 連続使用ストリークを確認する
**As a** 主婦ユーザーとして  
**I want to** 何日連続でDAMESIを使っているか確認したい  
**So that** 「堕落継続中」という達成感を感じられる  

**優先度**: Could

**Acceptance Criteria**:

```
Given ユーザーが献立を確定した
When 確定画面が表示される
Then 現在の連続使用日数が表示される（例：「3日連続！堕落継続中」）

Given ユーザーがトップ画面を開いた
When 画面が表示される
Then 現在のストリーク日数が表示される
```

---

### US-09: 週次レポートを確認する
**As a** 主婦ユーザーとして  
**I want to** 今週どれだけ思考を節約できたか確認したい  
**So that** DAMESIを使い続けるモチベーションになる  

**優先度**: Could

**Acceptance Criteria**:

```
Given 週が変わった（月曜日）
When ユーザーがアプリを開く
Then 「今週あなたが節約した思考時間：約XX分」という週次レポートが表示される

Given 週次レポートが表示されている
When ユーザーが「閉じる」を押す
Then レポートが閉じてトップ画面に戻る
```

## ドメイン 3: 献立ガチャ（FR-05）

### US-10: シングルガチャで献立を一発決定する
**As a** 主婦ユーザーとして  
**I want to** ガチャ演出で今日の献立を1品決めてもらいたい  
**So that** 意思決定を完全に放棄して楽しめる  

**優先度**: Must

**Acceptance Criteria**:

```
Given ユーザーがガチャ画面を開いた
When 「ガチャを回す！」ボタンを押す
Then ガチャ演出（スロット・カプセル回転など）が再生される

Given ガチャ演出が完了した
When 結果が表示される
Then レアリティ（N/R/SR/SSR）と料理名が表示される

Given ガチャ結果が表示された
When ユーザーが「これにする！」を押す
Then 献立が確定し、確定画面に遷移する（FR-02の確定フローに合流）
```

---

### US-11: リセマラを繰り返すと堕落ルートへ誘導される
**As a** 主婦ユーザーとして  
**I want to** 何度もリセマラしたときに面白い演出を見たい  
**So that** 笑いながらデリバリーという選択肢に気づける  

**優先度**: Must

**Acceptance Criteria**:

```
Given ユーザーがガチャ結果画面で「もう一度」を押した回数が4回以下の場合
When 「もう一度」を押す
Then 通常のガチャ演出が再生される

Given ユーザーがガチャ結果画面で「もう一度」を5回連続で押した
When 5回目のガチャ結果が表示される
Then メシストフェレスが登場し「…こっちにおいでよ」というセリフとデリバリーアプリへの誘導リンクが表示される
```

---

### US-12: 10連ガチャで1週間分の献立を一括決定する
**As a** 主婦ユーザーとして  
**I want to** 10連ガチャで1週間分の献立をまとめて決めてもらいたい  
**So that** 週1回の操作で1週間分の思考を放棄できる  

**優先度**: Should

**Acceptance Criteria**:

```
Given ユーザーがガチャ画面を開いた
When 「10連ガチャ」ボタンを押す
Then 10連ガチャ演出が再生され、7品の献立が一括で表示される

Given 10連ガチャ結果が表示された
When 結果を確認する
Then 7品のうち少なくとも3品はSR以上であることが保証されている

Given 10連ガチャ結果が表示された
When ユーザーが「この週の献立にする！」を押す
Then 7日分の献立が確定される
```

## ドメイン 4: AIキャラクター（FR-03, FR-06）

### US-13: 献立確定時にキャラクターの一言を受け取る
**As a** 主婦ユーザーとして  
**I want to** 献立を確定したときにキャラクターから一言もらいたい  
**So that** 毎回の操作が楽しくなり、アプリを使い続けたくなる  

**優先度**: Should

**Acceptance Criteria**:

```
Given ユーザーが献立を確定した（ワンボタン提案・ガチャ問わず）
When 確定画面が表示される
Then キャラクターのアイコンと一言セリフが表示される

Given 確定画面が表示された
When キャラクターの一言が表示される
Then 状況（trigger・from）に応じたトーンのセリフが表示される
  （キャラクター選択・トーン選択はキャラクタードメイン内部で決定）

Given 確定画面が表示された
When 同じ状況で複数回確定する
Then 毎回同じキャラクターが出るとは限らない（ランダム性がある）
```

---

### US-14: 料理登録時にキャラクターの一言を受け取る
**As a** 主婦ユーザーとして  
**I want to** 料理を登録したときにキャラクターから褒めてもらいたい  
**So that** 登録作業が楽しくなり、レパートリーを増やすモチベーションになる  

**優先度**: Should

**Acceptance Criteria**:

```
Given ユーザーが料理を登録した
When 登録完了画面が表示される
Then キャラクターのアイコンと登録を祝う一言セリフが表示される
```

---

### US-15: プレミアム機能で推しキャラを固定する
**As a** プレミアムユーザー（管理者にアンロックされたユーザー）として  
**I want to** 好きなキャラクターだけが一言を発するように設定したい  
**So that** 推しキャラへの愛着・沼りを深められる  

**優先度**: Could

**Acceptance Criteria**:

```
Given ユーザーのisPremiumフラグがtrueの場合
When 設定画面を開く
Then 「推しキャラ設定」メニューが表示される

Given 「推しキャラ設定」画面が表示されている
When ユーザーが1体以上のキャラクターを選択して保存する
Then 以降の一言は選択したキャラクターのみから発せられる

Given ユーザーのisPremiumフラグがfalseの場合
When 設定画面を開く
Then 「推しキャラ設定」メニューは表示されない
```
